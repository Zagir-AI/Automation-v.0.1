# UI Audit — elec-system/ui/app.py

Аудит проведён: 2026-05-12. Файл: `elec-system/ui/app.py` (~1185 строк).

---

## P1 — Критично (ломает функцию)

- [x] **UnicodeEncodeError при генерации документов на Windows**
  `run_cli()` строки 61–70: `subprocess.run()` без `encoding="utf-8"` — cp1251 не умеет ✓ ✗.
  _Фикс: добавлен `encoding="utf-8", errors="replace"` в subprocess.run. В PR #2._

- [x] **Нет кнопки "← К проектам" из экрана проекта**
  Строки 232–258: в шапке проекта нет способа сбросить `active_project` и вернуться на дашборд.
  Кнопки в сайдбаре открывают другой проект, но не дашборд. Единственный выход — перезапуск UI.
  _Фикс: добавить кнопку "← К проектам" в col_title, которая делает `st.session_state["active_project"] = None; st.rerun()`._

- [ ] **Дашборд: карточки проектов не отображаются**
  Строки 131–215: карточки формируются, но не видны — вероятно перекрыты вкладками проекта.
  Нужно изолировать дашборд через `st.empty()` или проверить порядок рендеринга.

- [x] **Нечётное количество проектов — последняя карточка не рендерится**
  Строки 167–170: `zip(cols, pair)` для неполной последней пары (1 проект вместо 2) не рендерит карточку.
  _Фикс: использовать `itertools.zip_longest` или заполнить None._

- [x] **Пустой список потребителей блокирует редактор кабеля**
  Строки 425–441: если удалить всех потребителей — `consumer_ids=[]` и редактор кабеля скрывается.
  Нельзя добавить новых. _Фикс: разделить логику — редактор кабеля независим от наличия потребителей._

## P2 — Важно (мешает работе)

- [x] **KeyError / TypeError в счётчике нарушений на дашборде**
  Строки 148–151: `cable_checks.get("du_violations", [])` может вернуть не список → TypeError.
  _Фикс: добавлен `_safe_len()` с `isinstance(..., list)` проверкой._

- [x] **Валидация кода и названия нового объекта**
  Строки 103–123: `if new_code and new_name:` пропускал пробелы и спецсимволы.
  _Фикс: regex `^[А-ЯA-Z0-9\-]{3,20}$` на код._

- [x] **Потеря cable при сохранении потребителей**
  Строки 493–508: если потребитель не найден в new_consumers, его cable не обновлялся, ошибка молчала.
  _Фикс: `st.error()` когда `target is None`._

- [x] **ValueError при конвертации типов в data_editor**
  Строки 471–491: `float(row.get(...))` без try-except. Если пользователь введёт текст — crash.
  _Фикс: обернуть в try-except, показать ошибку._

- [x] **IndexError / StopIteration при выборе щита**
  Строки 531–568: `next(...)` без дефолта — если selectbox вернёт значение вне panel_options → crash.
  _Фикс: `next(..., (None, None))` с проверкой на None._

- [x] **StopIteration при добавлении щита когда нет фидеров**
  Строка 558: `next(i for i, f in enumerate(vru["feeders"]) if ...)` без дефолта.
  _Фикс: `next(..., None)` с проверкой и `st.error()`._

- [x] **Нет try-except при расчёте тока КЗ**
  Строки 832–836: `calc_isc_from_tp(...)` — если tp_s_nom=0 → ZeroDivisionError → crash.
  _Фикс: обернуть в try-except с `st.error()`._

- [ ] **Несогласованность состояния при ошибке JSON**
  Строки 879–886: при JSONDecodeError переменная `project` в памяти остаётся старой,
  но UI продолжает работу со старыми данными незаметно для пользователя.
  _Фикс: явно откатить к последнему сохранённому состоянию или заблокировать действия._

- [x] **Молчаливый отказ при пустых feeders в результатах**
  Строки 905–929: если `vru_r.get("feeders", [])` пуст — ничего не выводилось.
  _Фикс: `st.info("Щиты не найдены в результатах расчёта. Нажми «Пересчитать всё».")`._

- [x] **None значения в таблице нарушений кабелей**
  Строка 991: `cb.get("isc_ka_source")` мог быть None → таблица неинформативна.
  _Фикс: `cb.get("isc_ka_source", "—")` и аналогично для section_mm2, kz_thermal_s_min_mm2._

- [x] **Нет try-except при парсинге файла сметы**
  Строки 1069–1072: повреждённый файл → необработанное исключение → crash UI.
  _Фикс: `try...except Exception as e: st.error(...)`._

- [x] **Накопление временных файлов при загрузке сметы**
  Строки 1064–1067: `NamedTemporaryFile(delete=False)` — файлы остаются на диске.
  _Фикс: использовать `delete=True` или `finally: tmp_path.unlink()`._
